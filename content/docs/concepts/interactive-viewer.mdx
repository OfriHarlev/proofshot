# The Interactive Viewer

The `viewer.html` file generated by `proofshot stop` is a self-contained HTML viewer — no external dependencies, no server required. Open it in any browser to review a verification session.

## Layout

```
┌─────────────────────────────────────────────────────┐
│  Header: description, console/server error badges   │
├──────────────────────────┬──────────────────────────┤
│                          │                          │
│  Video panel (62%)       │  Timeline panel (38%)    │
│                          │                          │
│  Custom scrub bar        │  Action steps with       │
│  with action markers     │  timestamps and icons    │
│                          │                          │
│  Overlay layer:          │  Click any step to       │
│  - Click ripples (blue)  │  seek the video          │
│  - Scroll indicators     │                          │
│  - Action label toasts   │  Arrow key navigation    │
│                          │                          │
└──────────────────────────┴──────────────────────────┘
```

## Features

**Scrub bar with action markers.** The progress bar shows markers at the exact timestamp of each action. Click a marker to jump to that moment in the video.

**Action timeline.** The right panel lists every action logged during the session — each with an icon (camera, mouse, keyboard, eye, compass), timestamp, and description. Clicking a step seeks the video to that moment.

**Video overlays.** A transparent layer renders on top of the video:
- **Click ripples** — blue circles with 600ms animation at the click location
- **Scroll indicators** — directional arrows with 800ms animation
- **Action toasts** — text labels at the bottom of the video describing the current action

Overlays are scaled from the original viewport size to the current video display size. A toggle checkbox lets you show/hide them.

**Keyboard navigation.** Left and right arrow keys jump between action markers on the scrub bar.

**Error badges.** Top-right corner displays console and server error counts. Green means clean, red means errors were found.

**Sync.** Playing the video highlights the current action in the timeline and auto-scrolls it into view. The two panels stay in lockstep.

**Responsive.** On smaller screens, the video and timeline stack vertically.

## How overlays work

The viewer embeds `session-log.json` data directly in a `<script>` tag. A `requestAnimationFrame` loop checks the current video time against action timestamps and renders overlays at the correct positions.

Element bounding boxes (captured during `proofshot exec`) are scaled from the original viewport dimensions to the current video element size. This means overlays stay aligned regardless of how large or small the viewer window is.

## Standalone and portable

The viewer has zero external dependencies. All CSS and JavaScript are inline. You can:

- Open it locally in any browser
- Attach it to a GitHub issue or PR
- Share it via Slack or email
- Host it on any static file server
